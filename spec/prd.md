# PolyMatrix Engine: 产品需求文档 (PRD)

### 1. 产品愿景与定位

PolyMatrix Engine 是一个专为 Polymarket 设计的自动化高频做市与统计套利系统。本系统摒弃传统预测市场的单向对赌思维，将每一个预测事件视为二元期权。通过构建合成价差组合、提供网格化流动性（Maker）以及高可用执行引擎，系统旨在零手续费环境下，通过概率错位和平台流动性补贴实现稳定、低风险的规模化盈利。

### 2. 核心商业目标

* **赚取 Maker 返利与流动性补贴**：系统将 80% 的时间作为流动性提供者（Maker），在合理定价区间内铺设双向网格，被动成交并获取官方补贴。
* **统计套利与概率修正**：捕捉极端情绪导致的短期定价偏离，在事件概率发生“坍缩”前完成低风险对冲。
* **冲击 Builder Program**：通过高频、稳定的 API 交互和庞大的 Maker 交易量，争取 Polymarket 官方开发者计划的资金与流量支持。

### 3. 核心功能模块 (Epic & User Stories)

#### 3.1 市场数据中心 (Market Data Hub)

* **实时订单簿监听**：通过 WebSocket 订阅目标市场的 Orderbook，纳秒级更新买卖盘口深度。
* **事件元数据同步**：通过 Gamma API 定期同步高流动性热点市场（如政治、宏观经济）的最新状态和决议结果。

#### 3.2 动态定价与做市引擎 (Pricing & Maker Engine)

* **基准概率锚定**：系统需支持输入外部数据源（如传统博彩赔率、大模型资讯分析）作为事件的基准概率。
* **动态网格铺单 (Grid Quoting)**：围绕基准概率，根据设定的安全边际和资金分配策略，自动在买卖双边生成梯次排列的限价单。
* **自适应撤单与重置**：当市场中间价发生偏移或订单被部分吃掉时，系统自动撤销旧订单并按新中轴重新铺设网格。

#### 3.3 风险与库存控制 (Risk & Inventory Management)

* **Delta 敞口监控**：实时计算在所有市场中的单向风险敞口（持有过多 Yes 或 No 份额）。
* **自动熔断与对冲**：当某一方向敞口达到预设阈值（如总资金的 15%），立即暂停该方向的 Maker 挂单，并触发市价（Taker）对冲指令以平抑风险。

#### 3.4 双通道执行网关 (Dual-Channel Execution Gateway)

* **常规免 Gas 通道**：日常挂单、撤单走 Polymarket 官方 Builder Relayer，实现零摩擦成本。
* **紧急逃生通道 (Kill Switch)**：监测到市场剧烈波动时，绕过 Relayer，直接通过私有 RPC 节点（如 Alchemy）向智能合约发送高 Gas 费用的紧急撤单指令，防止被抢跑。
